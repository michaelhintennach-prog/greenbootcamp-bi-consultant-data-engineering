# 🐍 Python Basics  
The *Python Basics* module introduces the fundamental concepts of the Python programming language and forms the foundation for data‑driven applications, automation, and later data‑engineering workflows.  
The content is divided into four main sections:

---

## 1. General

### **Bash Basics**  
Introduction to the command line, including navigation, file management, and process handling.  
**Relevance:** Data engineers frequently work with terminal tools, scripts, and automated processes.

### **Best Coding Practices**  
Clean code, meaningful variable names, structure, comments, and PEP8 conventions.  
**Relevance:** Readable code is essential for teamwork and long‑term maintainability.

### **Intro to Jupyter Lab**  
Working with Jupyter Notebooks: cells, Markdown, code execution, and visualization.  
**Relevance:** Ideal for exploratory data analysis and prototyping.

---

## 2. Basics

### **Numeric Variable Types**  
Working with `int`, `float`, type conversions, and mathematical operations.  
**Relevance:** Fundamental for calculations, aggregations, and data validation.

### **Strings**  
String manipulation, slicing, formatting, and common methods (`split`, `replace`, `upper`).  
**Relevance:** Many data sources contain text that must be cleaned and transformed.

### **If‑Statements**  
`if`, `elif`, `else` for controlling program flow.  
**Relevance:** Decision logic in ETL pipelines and data validation.

### **Loops**  
`for` and `while` loops, iterating over lists, dictionaries, and files.  
**Relevance:** Processing datasets and performing batch operations.

### **Plotting**  
First steps with visualization, e.g., using `matplotlib`.  
**Relevance:** Useful for exploratory data analysis and quality checks.

---

## 3. Data Structures in Python

### **Lists**  
Dynamic sequences, indexing, slicing, and common methods (`append`, `pop`).  
**Relevance:** The most frequently used structure for data processing.

### **Sets**  
Collections without duplicates, intersections, unions.  
**Relevance:** Ideal for data cleaning and identifying unique values.

### **Mutability**  
Mutable vs. immutable data types (e.g., lists vs. tuples).  
**Relevance:** Important for memory behavior and function design.

### **Dictionaries**  
Key‑value structures, access, iteration, and nesting.  
**Relevance:** Basis for JSON‑like data, API responses, and configurations.

### **Comprehension**  
List, set, and dictionary comprehensions for compact and efficient transformations.  
**Relevance:** Efficient data preparation in ETL workflows.

---

## 4. Functions

### **Introduction to Functions**  
Why functions matter: reusability, structure, modularization.

### **Function Definitions**  
Parameters, return values, default parameters, and scope.

### **Calling Functions**  
Function calls, passing arguments, and returning results.

### **Function Challenges**  
Practical exercises: data cleaning, transformations, mathematical operations.  
**Relevance:** Functions form the foundation of modular ETL pipelines and automated workflows.
